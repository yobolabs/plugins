### What a good `/gtm` run looks like (from the Northwind exemplar)

- **Phase 1 brief names the buyer question as the central UNRESOLVED bet** when the founder is unsure — it does not force a motion. Three candidate motions (PLG self-serve / founder-led design partners / marketplace-integration listing) framed as competing hypotheses for the spar to kill.
- **Load-bearing bets written as falsifiable claims**, not vibes — that is what makes them sparrable.
- **The spar kills motions on evidence** (marketplace-listing-now eliminated: zero inbound + no sales force + short runway + long marketplace-review base rates), and **reshapes founder ideas rather than validating or dismissing** (the passive marketplace pull-detector challenged; a compliant founder-led design-partner motion derived).
- **The one fact that reorders everything is hunted first** (external willingness-to-pay from on-call teams) and the ranking is honest about being contingent on it.
- **War-game turns the surviving strategy's irreversible moves into guardrails** — public-repositioning commit, pricing-model commit, narrow-not-broad positioning — with abort conditions tied to a falsifiable tripwire (paid design-partner conversions inside the smoke-test window), not vibes.
- **Synthesis = strategy.md (bets tagged survived/amended/killed + evidence) + decision-ledger.md (decision · assumption · reversibility · flip observable).** Blocked facts (revenue target, runway, team) surfaced as ASSUMED, never silently defaulted.

## Rules for adding exemplars

- Only from full runs that reached GATE 2 (or a documented honest abort).
- Full files → `SPAR_HOME/exemplars/`; this index gets the pointer + behavior notes.
- Note provenance (framework-run vs frontier-direct) and, if executed, what reality diverged on — never edit an exemplar to look better than the run was.
