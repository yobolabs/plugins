# Capability Gap — honest residue log

What still needs a frontier model. Every calibration miss classified `capability` (not procedure, not knowledge — those get patched into SKILLs/packs/traps) lands here as a permanent, honest entry. An empty file before calibration has run is correct; a file that stays empty after calibration is suspicious.

**Entry schema (specs §6.6):**

```
## GAP-NNN — <one-line behavior observed, generalized>
- **Mission class:** <audience class + placement set + care level — never a named merchant>
- **Classification:** capability
- **Observed:** <what the framework run missed that the frontier-direct run caught, generalized>
- **Patch attempted:** <procedure/knowledge patch tried and found insufficient, or "none — inherently frontier">
- **Residue statement:** <why this is capability, not retrievable knowledge or fixable procedure>
```

Redaction rule: same as exemplars — no estate org/merchant names, no raw campaign numbers (validator privacy scope).
