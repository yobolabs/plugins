# Exemplars — Fable-direct ground truth

One entry kind: **Fable-direct run distillations** — accepted spec+mockup pairs from frontier-model-direct app-design missions, distilled and redacted. These are the judge's calibration anchors: what "good" looked like, direction chosen, distilled judge standards, usability-argument quality notes, outcome classes only. New entries only from models at least that strong.

**Self-containment rule:** every entry stands alone — a judge briefed with only this directory must be able to apply the standards. `_context/` pointers are optional, repo-relative, marked `local-only`; a missing pointer never blocks anything.

**Redaction rule:** no real end-user/customer data, no raw usage numbers (validator privacy scope). Ecosystem app names, screen names, and component/token names are fine (D7). Mission classes only (screen class + persona class + care level).

**Naming:** `fable-<mission-class-slug>.md`.
